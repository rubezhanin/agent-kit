# AGENT.md - настрой короткий входной буфер сообщений

## Цель

Настроить агента/gateway так, чтобы сообщения, которые пользователь отправил почти одновременно, попадали агенту одним общим входом, а не обрабатывались по одному.

Типичный случай:

```text
Пользователь отправил:
1. текст задачи
2. файл
3. уточнение

Агент должен получить один turn:
текст задачи + файл + уточнение
```

Это не память агента. Это короткая задержка на входе перед стартом обработки.

## Главное правило

Собирай только сообщения, которые пришли одному агенту почти одновременно от одного пользователя в одном чате/топике.

Не объединяй:
- разных пользователей;
- разные чаты;
- разные топики/threads;
- сообщения, пришедшие сильно позже;
- новую задачу с уже выполняемой задачей.

## Рекомендуемое поведение

1. Когда пришло первое сообщение, не запускай агента сразу.
2. Открой короткое окно ожидания: `0.8-1.5 сек`.
3. Все сообщения от того же пользователя в том же чате/топике, пришедшие в это окно, сложи в один пакет.
4. Текст объедини по порядку через перенос строки.
5. Файлы добавь в общий список `attachments`.
6. Картинки добавь в общий список `images/media`.
7. Сохрани порядок сообщений.
8. После flush передай агенту один общий turn.
9. Если агент уже занят, новый пакет поставь в очередь, а не перебивай текущую работу.

## Session key

Буфер должен быть разделён по ключу:

```text
platform + agent/profile + chat_id + thread_id/topic_id + user_id
```

Если в платформе нет topic/thread, используй `null` или пустое значение, но не выбрасывай поле из логики.

## Тайминги

Стартовые значения:

```text
debounce window: 0.8 сек
hard cap: 1.5-2.0 сек
busy mode: queue
```

Не ставь 10-30 секунд. Агент станет ватным.

## Если есть готовая настройка

Сначала ищи config-путь. Не лезь в код, если gateway уже умеет batching/debounce.

Для Hermes смысл может выглядеть так, если текущая версия поддерживает эти ключи:

```yaml
platforms:
  telegram:
    extra:
      ingress_batch_seconds: 0.8
      ingress_batch_hard_cap_seconds: 1.5

display:
  busy_input_mode: queue
```

Не вставляй этот YAML вслепую. Проверь реальные имена настроек в текущей версии.

## Если настройки нет

Добавь безопасный механизм:

```text
on_incoming(event):
  key = platform + agent + chat_id + thread_id + user_id
  buffer = buffers[key]
  buffer.add(event)
  buffer.flush_at = now + debounce_window
  buffer.hard_deadline = first_event_time + hard_cap

flush(key):
  package.text = join_texts_in_order(buffer.events)
  package.attachments = collect_files_in_order(buffer.events)
  package.images = collect_images_in_order(buffer.events)
  create_agent_turn(package)
  log("ingress_batch_flush", key_hash, item_count, file_count, image_count)
```

Логи без секретов. Не печатай токены, `.env`, OAuth, cookies, Telegram token, session files.

## Тесты

Проверь минимум:

1. Текст + файл почти одновременно -> один turn.
2. Текст + картинка почти одновременно -> один turn.
3. Три текстовых сообщения подряд -> один turn, порядок сохранён.
4. Два файла подряд -> один turn, оба файла в attachments.
5. Два пользователя одновременно -> разные turns.
6. Два чата или топика -> разные turns.
7. Сообщение пришло после hard cap -> новый turn.
8. Агент занят -> новый пакет в очереди, текущая задача не оборвана.

## Что должно быть в логах

```text
ingress_batch_add type=text session=...
ingress_batch_add type=file session=...
ingress_batch_flush items=2 text_parts=1 attachments=1
agent_turn_created source=batch
```

Названия могут отличаться. Смысл: один flush и один agent turn.

## Нельзя

- Не лечи это промптом. Промпт не решает гонку входящих событий.
- Не смешивай разных пользователей/чаты/топики.
- Не ставь большое окно ожидания.
- Не перезапускай всю систему, если достаточно одного gateway/profile.
- Не трогай секреты.
- Не печатай содержимое `.env`.
- Не делай внешних действий без подтверждения человека.

## Отчёт после настройки

Верни короткий отчёт:

```text
Сделано:
- где найден gateway/turn creation;
- какие файлы изменены;
- какая настройка включена или какой механизм добавлен;
- какие тесты прошли;
- как проверить вручную;
- как откатить.

Проверка:
- текст + файл -> один turn: да/нет;
- текст + картинка -> один turn: да/нет;
- разные пользователи не смешались: да/нет;
- разные чаты/топики не смешались: да/нет;
- busy mode queue работает: да/нет.
```
