# Package Contract

Этот пакет должен быть переносимым и безопасным для участников.

## Разрешено

- использовать свои данные о бизнесе, продукте и аудитории;
- использовать свои цвета, референсы, брендбук, фото и стиль;
- создавать собственные identity/style packs;
- генерировать prompts для GPT Image;
- использовать собственную совместимую подписку/доступ, если локальный инструмент это поддерживает;
- адаптировать шаблоны под свою нишу.

## Запрещено

- вставлять реальные API keys, access tokens, cookies, session files или OAuth-данные в Markdown;
- просить пользователя отправить секреты в чат;
- обещать гарантированный одинаковый результат генерации;
- использовать чужие лица, логотипы, брендбуки или закрытые материалы без прав;
- выдавать медицинские, финансовые, юридические или карьерные гарантии без проверки;
- публиковать prompts/receipts, если там есть приватные данные клиента;
- использовать закрытые style packs, face references или внутренние рабочие материалы автора пакета;
- включать абсолютные локальные пути автора пакета.

## Позиционирование

Правильная формулировка:

```text
Это переносимый workflow для планирования и производства каруселей: анкета бренда, SMM-brief, структура, визуальное ТЗ, prompt-pack и OTK. Генерация изображений опциональна и использует инструменты/доступ самого участника.
```

Неправильная формулировка:

```text
Одна кнопка автоматически делает идеальные карусели через чужую подписку.
```

## Безопасный дефолт

Любой агент, использующий этот kit, должен сначала собрать `brand-profile.md`. Если данных мало, он задаёт вопросы или явно пишет `Assumptions`, а не выдумывает бизнес пользователя.
