# 03 — Carousel Workflow

## Общая цепочка

```text
User idea
-> Brand Intake
-> brand-profile.md
-> SMM/Marketing Brief
-> carousel-brief.md
-> slide-plan.md
-> Visual Designer
-> visual-brief.md
-> prompt-pack.md
-> GPT Image / designer / manual layout
-> OTK
-> final-delivery-report.md
```

## Фаза 0 — не генерировать

Сначала собрать:

- кто пользователь;
- что продаёт/объясняет;
- кому;
- зачем;
- где публикует;
- какой стиль;
- какие ограничения.

## Фаза 1 — смысл

Сформировать:

- боль;
- hook;
- обещание;
- структуру;
- один смысл на слайд;
- CTA;
- engagement mechanic.

## Фаза 2 — визуал

На каждый слайд:

```text
Slide N
Text:
Meaning job:
Visual metaphor:
Composition:
Text zone:
Style:
Prompt:
Negative constraints:
```

## Фаза 3 — generation plan

Перед генерацией:

- проверить prompt-pack;
- убрать слишком длинный текст;
- проверить, не требуется ли лицо/референсы;
- понять, какой формат нужен;
- предупредить о рисках текста/лица/рук.

## Фаза 4 — финальный OTK

Проверить:

- читаемость на телефоне;
- cover;
- slide 2;
- последовательность;
- CTA;
- риски;
- стиль;
- артефакты;
- готовность к публикации.

## Форматы

### Telegram

```text
Square: 1280x1280 или 1024x1024
Landscape: 1280x720 или 1920x1080
Portrait: 1080x1350, если переиспользуется Instagram
```

Telegram default: один крупный смысл, большой заголовок, минимум мелкого текста.

### Instagram

```text
Canvas: 1080x1350
Aspect: 4:5
Safe margin: 10-12%
Cover must survive central 1:1 crop
```
