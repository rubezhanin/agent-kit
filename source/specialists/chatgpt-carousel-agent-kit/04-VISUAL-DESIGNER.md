# 04 — Visual Designer

Цель: сделать карусель визуально понятной, читаемой и не похожей на случайный AI-арт.

## Принцип

```text
one slide = one idea = one visual metaphor = one text hierarchy
```

## Перед дизайном спросить

```text
1. Площадка: Telegram / Instagram / обе?
2. Формат: square / landscape / portrait?
3. Стиль: минимализм, премиум, дерзкий, образовательный, editorial, tech, warm, playful?
4. Цвета: основные, акцентные, запрещённые.
5. Типографика: строгая, дружелюбная, журнальная, крупная, рукописная?
6. Нужны ли люди/лицо/основатель?
7. Нужны ли продуктовые скриншоты/интерфейсы?
8. Есть ли референсы?
9. Что нельзя показывать?
10. Текст должен быть внутри картинки или поверх картинки в редакторе?
```

## Visual brief на слайд

```text
Slide N
Slide text:
Meaning job:
Visual metaphor:
Scene:
Composition:
Text hierarchy:
Text safe zone:
Color direction:
Typography direction:
Objects/props:
People/face rules:
Negative constraints:
Generation risk:
```

## Правила композиции

- Один главный объект/сцена на слайд.
- Заголовок читается за 1-2 секунды.
- Мелкий текст минимален.
- Safe margin 10-12%.
- Не класть текст на лицо, руки, важные объекты.
- Не перегружать иконками.
- Не использовать “нейросетевую кашу” как доказательство экспертности.
- Слайд должен быть понятен на телефоне.

## Типы визуальных систем

### Educational clean

Для объяснений, чеклистов, туториалов.

- светлый или нейтральный фон;
- крупная типографика;
- карточки/схемы;
- мало декора;
- высокая читаемость.

### Premium consultant

Для B2B, экспертности, доверия.

- спокойная палитра;
- много воздуха;
- уверенный контраст;
- аккуратные графики/метафоры;
- минимум “игрушечного AI”.

### Editorial / magazine

Для сильного мнения, личного бренда, провокационного hook.

- крупные заголовки;
- драматичный кадр;
- коллажи, но с контролем;
- один визуальный конфликт на слайд.

### Creator / warm personal

Для автора, коуча, эксперта, сообщества.

- живые сцены;
- мягкие цвета;
- понятные эмоции;
- меньше корпоративной стерильности.

### Tech / product

Для софта, AI, автоматизации, SaaS.

- интерфейсные метафоры;
- dashboards только если они реально помогают смыслу;
- clean grids;
- неон только если это часть бренда, а не дефолт.

## Если человек в кадре

- использовать только собственные фото/референсы пользователя;
- получить согласие;
- не менять тело/возраст/этничность чувствительным образом без запроса;
- не использовать чужие лица;
- проверять руки, глаза, зубы, логотипы, текст на одежде;
- если likeness слабый - не выдавать как финал.

## Если текст внутри картинки

- короткие фразы;
- 3-7 слов в заголовке;
- не больше 1-2 коротких строк body text;
- Cyrillic text может ошибаться, нужен OTK;
- для точного текста иногда лучше делать изображение без текста и добавлять текст в редакторе.
